## Architecture
Use:
- Clean Architecture
- Repository Pattern
- Feature-based folder structure

## State Management
Recommended:
- Riverpod
or
- Bloc

## Backend
Use Supabase for:
- Auth
- Database
- Storage

## Database
Use PostgreSQL schema.

## UI Style
- Minimalist
- Modern
- Comfortable reading experience
- Dark mode support

# Expected Output

Generate:
- Folder structure
- Flutter code
- Models
- Services
- Repositories
- UI pages
- Widgets
- API integration
- Database setup

- Avoid unnecessary complexity.
- Write readable code.
- Use comments only when necessary.
- Separate business logic from UI.
- Keep widgets modular.
